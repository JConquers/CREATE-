Abstract—Transformer architectures have become the dominant approach in sequential recommender systems due to their
ability to capture dependencies within user interaction histories.
However, these models often consider sequence elements in
isolation, failing to fully leverage the global collaborative context
inherent in user–item relationships. While recent frameworks like
CREATE [1] address this by aligning sequential encoders with
graph neural networks (GNNs), they remain largely limited to
unsigned bipartite graphs that cannot model explicit negative
feedback or higher-order multi-way relationships. To fill this
gap, we propose two architectural extensions: CREATE-Pone and
CREATE-Uni. By integrating signed graph modeling inspired
by Pone-GNN [2] to disentangle interest from disinterest and
hypergraph structures via the UniGNN [3] framework to capture
complex higher-order correlations, our framework provides a
more expressive and nuanced grounding for sequential models. This dual approach significantly enriches the structural
knowledge transferred to the transformer while preserving the
efficient asymmetric inference protocol required for productionscale recommendation.
Index Terms—Recommender Systems, Sequential Recommendation, Signed Graphs, Hypergraphs

I. I NTRODUCTION
Modern recommender systems must reconcile two distinct
perspectives of user behavior: short-range sequential dynamics
and long-range collaborative structures. While Graph Neural
Networks (GNNs) have emerged as the state-of-the-art for capturing global collaborative signals through recursive neighborhood aggregation, they are often less effective at modeling the
strictly ordered, temporal dependencies inherent in user interaction histories. Conversely, Transformer-based architectures
excel at capturing these fine-grained sequential patterns but
struggle to encode the broader structural context of the global
user–item interaction graph. To bridge this gap, the recently
proposed CREATE framework [1] introduces a paradigm of
cross-representation knowledge transfer. By using a GNN as
a global structural grounding to enrich a transformer-based
sequential encoder during training, CREATE achieves superior
performance by combining the strengths of both architectures.
This alignment allows the model to inherit global collaborative
wisdom while maintaining an efficient asymmetric inference
protocol that utilizes only the sequential encoder at test time.
Despite its effectiveness, the original CREATE framework
relies on unsigned bipartite graphs, which introduce two
significant structural bottlenecks. First, it suffers from polarity
blindness; by treating all observed interactions as positive, it
ignores explicit negative feedback (e.g., low ratings or skips),

leading to label noise and an inability to model user aversion.
Second, it is limited by pairwise collapse, where higherorder multi-way relationships—such as a user’s entire session
history or co-purchase groups—are reduced to isolated binary
links. This simplification results in a loss of complex collaborative context that pairwise edges cannot distinguish. Third,
the framework is hindered by exposure bias. Because unsigned
bipartite GNNs typically treat all unobserved interactions as
negative signals, the model fails to distinguish between items a
user actively disliked and those they simply never encountered,
leading to suboptimal generalization.
In this paper, we propose two extensions to the CREATE
framework to recover this lost structural information:
CREATE-Pone (Signed Variant): We replace the unsigned encoder with a signed GNN [2] that learns dual
interest and disinterest embeddings. The alignment framework is extended to a multi-objective task: the sequential
user representation is aligned with the global interest view
to maximize knowledge transfer, while simultaneously
being constrained to be orthogonal to the disinterest view.
This allows the transformer to inherit collaborative preferences while explicitly suppressing signals associated with
user aversion.
• CREATE-Uni (Hypergraph Variant): We leverage a
unified hypergraph framework [3] to model user histories as hyperedges. This transition enables the model
to capture multi-way collaborative signals with 1-GWL
expressive power, strictly exceeding the topological limits
of traditional bipartite GNNs like LightGCN.
•

Our contributions include the formalization of these signed
and higher-order alignment objectives, a theoretical analysis of
their expressive advantages, and an extensive empirical evaluation across five benchmark datasets. The remainder of this
paper is organized as follows: Section II reviews related work,
Section III presents the methodology, Section IV provides
theoretical analysis, and Section V describes the experimental
setup.
II. R ELATED W ORK
The evolution of recommendation systems has moved from
modeling isolated sequences to capturing global structural
dependencies, and recently, toward the alignment of these two
distinct paradigms.

A. Sequential Recommendation
Sequential recommendation focuses on predicting the next
item based on a user’s chronological interaction history.
Early models utilized Recurrent Neural Networks (RNNs),
but the field has been revolutionized by Transformer-based
architectures. SASRec [4] adopted a self-attention mechanism
to identify relevant items in a sequence, while BERT4Rec
[5] introduced bidirectional modeling via masked language
objectives. To improve robustness, CL4SRec [6] integrated
contrastive learning to ensure that different augmentations of
the same sequence yield similar representations. However,
these models primarily focus on local dependencies and often
lack a global perspective of item-item collaborative signals.
B. Graph-based Collaborative Filtering
Graph Neural Networks (GNNs) explicitly model user–item
interactions as a bipartite graph to capture long-range collaborative signals. This paradigm involves learning d-dimensional
node embeddings for both users (eu ) and items (ei ) by
aggregating information from their respective neighborhoods.
NGCF [7] first introduced neighborhood aggregation to recommendation, but was later simplified by LightGCN [8], which
removed non-linear activations and weight matrices to improve
efficiency and performance:

bottleneck, modern hybrid systems utilize sequential models
for inference, relying on the GNN only during the training
phase to provide an enriched, structural grounding for the
sequential item embeddings.
C. Cross-Representation Knowledge Transfer
The CREATE framework [1] serves as the primary basis
for this work, bridging the gap between local sequential patterns and global collaborative structures through representation
alignment. The framework utilizes a dual-encoder architecture:

a sequential encoder (such as SASRec [4] or BERT4Rec [5])
processes the interaction history to generate a temporal user
representation hu , while a graph encoder (typically LightGCN
[8] or UltraGCN) processes the global bipartite graph to
produce structural user embeddings zu and item embeddings
zi .The core innovation of CREATE is the use of the Barlow
Twins objective [9] to align these two distinct views. After
batch standardization of hu and zu , the model computes a
cross-correlation matrix C, where the alignment loss Lalign
forces the sequential representation to inherit structural context
by driving C toward the identity matrix:

E. Signed Graph Learning
Despite the success of alignment frameworks, most GNNbased recommenders operate on unsigned bipartite graphs,
treating all observed interactions as positive signals. This
introduces significant label noise and exposure bias, as the
model cannot distinguish between items a user actively disliked and those they simply never encountered. To address
this, Pone-GNN [2] introduces a signed graph framework
that explicitly models negative feedback by partitioning the

global interaction set E into two disjoint subsets: a positive
interaction graph Gp = (V, Ep ) and a negative interaction

message-passing branches to learn dual representations for
each node: an interest embedding z ∈ Rd derived from Gp ,
and a disinterest embedding v ∈ Rd derived from Gn . These
node embeddings are optimized via a contrastive learning
loss that ensures the latent spaces are well-separated and
informative. For the final recommendation task, the model
utilizes a dual feedback-aware loss LDF to maximize the
likelihood of observed interactions within their respective
polarities:

where Ou denotes items the user has explicitly liked (e.g.
ratings ≥ 4), Du denotes items the user has explicitly disliked

2) Limitation 1 — Exposure Bias in Unsigned BPR: Since
LightGCN folds Du into the positive graph, BPR samples
negatives exclusively from Su — items the user has simply
never been exposed to. However, Su contains no genuine
dislike signal: these are items the user may well enjoy but has
never encountered. In practice |Su | ≫ |Du | for most users,
so the model is trained to rank observed items above a vast
pool of potentially enjoyable unseen items, receiving noisy
and misleading supervision throughout.
This is the exposure bias: the true negative evidence in Du
is never used as such, while Su — which carries no genuine
negative signal — is treated as the sole source of negative
supervision.

Resolution in CREATE-Pone. The dual feedback loss LDF
(Eq. 9) draws negatives exclusively from Du ≡ En — items
the user has explicitly disliked. Items from Su are never used
as negative evidence, ensuring that every negative training
signal is semantically meaningful.
3) Limitation 2 — Polarity Blindness in LightGCN: Because LightGCN places all edges from Ou ∪ Du into a
single unsigned graph, it has no awareness that Du represents
negative feedback. Every item a user has rated — whether
liked or disliked — is treated as a positive neighbour and
contributes to the user embedding via message passing in
exactly the same way.
This is the polarity blindness of unsigned graphs. The user
embedding becomes an indiscriminate mixture of interest and
disinterest signals, with no mechanism to separate them. As a
result, items in Du actively pull the user representation toward
themselves during aggregation — the same direction as liked
items. Any candidate item j similar to a disliked item a ∈ Du
will therefore receive an inflated recommendation score via
the inner product ŷuj = e⊤
u ej , since a’s embedding has been
folded into eu as if it were a preference. The more disliked
items a user has interacted with, the more corrupted their
embedding becomes.
Resolution in CREATE-Pone. The dual-branch construction
propagates interest embeddings zu solely on Gp (built from
Ou ) and disinterest embeddings vu solely on Gn (built from
Du ). Since Gp and Gn share no edges, the two branches are
completely isolated — items from Du can never corrupt zu ,
and the interest signal is preserved cleanly.
4) Remark — Pull–Push Duality in Alignment: The alignment loss (Eq. 15) plays two complementary roles. The Barlow
Twins term ∥C hz − I∥2F → 0 acts as a pull, aligning the
sequential state hu toward the interest subspace of zu , so
the transformer inherits collaborative signals from Ou . The
orthogonality term µ∥C hv ∥2F → 0 acts as a push, driving
the cross-correlation between hu and vu to zero and thereby
preventing disinterest signals from Du from leaking into
sequential recommendations.
No such push exists in the original unsigned CREATE,
which operates on a single graph over Ou ∪Du and has no loss
term to stop the sequential encoder from absorbing aversion
signals. CREATE-Pone’s alignment is therefore strictly richer.
B. CREATE-Uni: Hypergraph Expressivity
1) Established Results We Rely On: We use three results
from the literature without reproving them:
• GNNs ≤ 1-WL [15]: Any message-passing GNN (including LightGCN) is at most as powerful as the 1dimensional Weisfeiler-Leman (1-WL) graph isomorphism test. Two nodes that 1-WL cannot distinguish will
receive identical embeddings from any such GNN.
• UniGNN ≡ 1-GWL [3]: When the aggregation functions
ϕ1 (node-to-hyperedge) and ϕ2 (hyperedge-to-node) are
injective on multisets, UniGNN is equivalent in power
to the 1-dimensional Generalized Weisfeiler-Leman (1GWL) test on hypergraphs.

•

1-GWL ≥ 1-WL [16]: 1-GWL is strictly more powerful
than 1-WL on structures containing multi-way relationships, such as hyperedges of size ≥ 2, which pairwise
graphs cannot natively represent.

What remains to be shown is that the specific user-modeling
scenario in CREATE-Uni actually instantiates the strict gap
— i.e., that there is a concrete, interpretable user pair that
LightGCN conflates but UniGNN separates.

4) Corollary — CREATE-Uni Provides Richer Alignment
Targets than CREATE: Statement. The Barlow Twins alignment loss Lalign (Eq. 22) conveys strictly more discriminative
gradient signal to the sequential encoder under CREATE-Uni
than under the original CREATE.
Proof sketch. By Proposition IV-B3, u1 and u2 receive
different hypergraph embeddings: gu1 ̸= gu2 . In a batch
containing both users, the batch-normalised targets g̃u1 ̸= g̃u2
contribute different columns to the cross-correlation matrix
C hg , so the alignment loss generates informative gradients
that push hu1 and hu2 apart.
Under LightGCN, zu1 = zu2 (Proposition IV-B2), so
both users produce identical alignment targets. The sequential encoder receives no gradient from Lalign to differentiate
hu1 from hu2 , even though their behavioral patterns differ.
The mutual information transferred via alignment is therefore
strictly greater in CREATE-Uni.
V. DATASETS
We evaluate our method on five widely used recommendation datasets: Amazon Beauty(amazon-reviews-2023), Amazon Books(amazon-reviews-2023), Yelp, Yambda-50M, and
KuaiRand-1K. All datasets are modeled as bipartite graphs
G = (U , I, E), where U denotes users, I denotes items, and
E denotes user–item interactions.
Amazon Reviews 2023 (Beauty, Books): In these datasets,
user nodes correspond to Amazon customers and item nodes
correspond to products (beauty products or books respectively). An edge (u, i) represents a user review or purchase
interaction with an item. Edge features include the review
rating and timestamp of the interaction. Item node features
are derived from product metadata such as category, title, and
textual description when available. User nodes typically do
not contain explicit features and are represented via learned
embeddings.

Yelp: In the Yelp dataset, user nodes represent Yelp users
and item nodes represent local businesses. Edges correspond
to user reviews of businesses. Edge attributes include rating
scores and timestamps. Item node features are constructed
from business metadata such as categories and attributes, while
user nodes are represented implicitly through their interaction
histories.
Yambda-50M: Yambda-50M is a large-scale implicit feedback dataset derived from Yahoo! music interactions. User
nodes represent listeners and item nodes correspond to music
tracks. Edges denote implicit listening events between users
and tracks. Edge features primarily consist of timestamps
indicating when the interaction occurred. No explicit user or
item features are provided; representations are learned directly
from the interaction graph.
KuaiRand-1K: KuaiRand-1K is a short-video recommendation dataset collected from the Kuaishou platform. User
nodes represent viewers and item nodes correspond to short
videos. Edges denote user exposure and interaction events such
as clicks or watches. Edge features include timestamps and
interaction signals (e.g., click or watch behavior). Item node
features include video metadata such as category and content
descriptors when available.
